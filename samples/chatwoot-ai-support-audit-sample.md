# Chatwoot + AI 客服技术审计与 MVP 方案样例

> **声明**：这是独立投递准备样例，基于 2026-07-16 可公开的一手资料制作。样例未受任何客户委托，与电鸭需求发布方没有客户关系，未访问其 Chatwoot 系统、源代码、配置、数据或内部文档，也没有声称任何 Chatwoot 二开客户经验。以下内容只展示分析方法，不构成对真实系统的审计结论。

## 公开需求摘要

电鸭项目页于 2026-06-20 发布。公开需求显示，现有环境包含 Chatwoot Enterprise 源码与测试环境、PostgreSQL、Redis、部分 AI 客服及翻译脚本和产品资料。第一阶段计划在 1 至 2 周内形成现状审计、保留/定制/独立建设矩阵、AI 客服与实时翻译方案、RAG/FAQ 知识库、无结果与人工接管、WhatsApp/Telegram/Messenger/LINE 渠道架构、30 天实施计划、风险及验收方案。

## 待确认问题

1. 当前 Chatwoot 版本、Enterprise 许可、部署拓扑、源码改动点、升级与回滚记录是什么？
2. 每日会话量、峰值并发、语言、渠道占比、客服人数、工作时段和目标 SLA 是什么？
3. 现有 Inbox、Team、Agent、Automation、Assignment、Label、SLA 和报表如何配置？
4. 现有 AI、翻译和 RAG 使用什么模型、端点、检索库、日志策略、数据保留期和成本上限？
5. 哪些资料属于唯一可信来源，谁负责审批、版本更新、下线和敏感信息清理？
6. 哪些场景必须人工处理，接管时限、值班责任、升级路径和对话上下文要求是什么？
7. 各渠道的企业主体、平台账号、凭证、消息模板、区域限制和审批状态是什么？
8. 成功指标、延迟、可用性、RPO/RTO、隐私边界、预算和第一个可上线业务范围是什么？

## 建议 MVP 架构

`单一渠道/Inbox → 身份与路由 → AI 策略门 → 已审批知识检索 → 回答或无结果 → 人工接管 → 质量与成本报表`

| 模块 | MVP 内容 | 关键取舍 |
|---|---|---|
| Chatwoot 基座 | 1 个 Inbox、团队分配、标签、自动化、SLA、报表 | 先验证一个真实业务域，暂不并行铺开全部渠道 |
| AI 接入 | 形成 Captain、Copilot、AgentBot 能力与成本矩阵 | 高风险场景采用 Copilot 辅助人工；低风险直答仅在规则和许可满足时启用 Captain 或 AgentBot |
| RAG | 一套已审批 FAQ/产品资料、版本号、来源链接、更新时间 | 不抓取来源不明或未经审批内容；每条答案保留来源与置信信号 |
| 策略门 | 风险分类、身份状态、语言、无结果、敏感意图和工具权限 | 默认最小权限；客户专属数据查询需要已验证身份 |
| 人工接管 | `pending → open` 接管、上下文摘要、原因标签、SLA | 显式要求人工、低置信、投诉、退款、账户变更均直接接管 |
| 安全与可观测 | 签名校验、重放防护、幂等、审计日志、成本/延迟/错误指标 | 日志脱敏；模型、工具、Webhook 失败时回退人工队列 |

**暂定架构决策 ADR-001**：第一版只覆盖一个 Inbox 和一个低风险知识域，所有高风险意图保留人工闭环。优点是范围可验收、回滚简单、能建立真实基线；代价是短期无法覆盖所有渠道和业务流程。是否采用 Captain、Copilot 或外部 AgentBot，须在版本、许可、数据流、人工审核要求与成本核验后决定。

## 人工接管与 RAG 边界

| 可进入自动回答候选 | 必须人工接管 |
|---|---|
| 已审批且稳定的公开产品说明、营业时间、通用流程、非账户型 FAQ | 用户明确要求人工、无检索结果、多个来源冲突、低置信或多轮仍未解决 |
| 不含个人数据、账户变更、付款决策的说明性问题 | 退款/账单、投诉、账户或权限变更、法律医疗财务意见、个人敏感信息 |
| 身份无关的公开政策，且答案能显示来源和更新时间 | 身份未验证却请求客户专属数据、疑似提示注入、工具异常、模型或翻译失败 |

RAG 只使用有负责人、有版本、有更新时间的批准语料。知识命中不能覆盖风险策略。客户专属工具调用须校验 Chatwoot 提供的身份验证上下文，并对工具响应进行清洗。Webhook 采用原始请求体执行 HMAC-SHA256 校验、时间戳窗口、常量时间比较和 Delivery ID 幂等控制。

## 第一周排期

| 日期 | 工作 | 当日产物 |
|---|---|---|
| D1 | 只读证据清单、版本/许可/拓扑、源码改动与访问边界访谈 | 现状图、证据缺口、访问清单 |
| D2 | 会话、渠道、数据流、角色、SLA 与非功能需求梳理 | 数据流图、风险分级、NFR 表 |
| D3 | 保留/定制/独立建设矩阵；RAG、人工接管和 AI 接入决策 | 方案矩阵、ADR、MVP 范围 |
| D4 | 在隔离测试环境完成 1 个 Inbox 的最小链路和 20 至 30 条测试场景 | 测试记录、失败回退演示，不接入真实客户流量 |
| D5 | 汇总缺陷、容量与成本假设、30 天待办、团队分工和回滚方案 | 审计报告、路线图、风险台账、评审会 |

## 验收清单

- [ ] 有证据链接的现状架构图，以及保留/定制/独立建设矩阵。
- [ ] 知识来源、负责人、更新/下线流程、无结果行为均可复核。
- [ ] 隔离环境验证 Bot 与人工的 `pending/open/pending` 状态切换，接管上下文不丢失。
- [ ] Webhook 签名、时间戳、重放攻击、重复投递和乱序场景均有测试记录。
- [ ] 高风险测试用例 100% 转人工，身份未验证时不返回客户专属数据。
- [ ] 模型、工具、Webhook、Redis、Worker 或渠道异常时能回退、告警和恢复。
- [ ] 能记录首响时间、解决时间、接管率、自动解决率、延迟、失败率与单会话 AI 成本。
- [ ] 部署、数据库迁移、回滚、备份恢复、权限与日志脱敏方案通过评审。

## 主要风险

| 风险 | 初步控制 |
|---|---|
| 版本或私有分支未知，升级可能破坏二开 | 先做差异清单、迁移预演和可回滚发布 |
| Captain 许可、付费计划、credits、BYOK 或模型端点条件不清 | 开工前核验版本、合同与单位会话成本，保留 AgentBot 方案 |
| 知识库陈旧、解析错误、来源冲突 | 语料审批、版本化、回归问集、到期复审和无结果转人工 |
| 模型数据保留、跨境传输和隐私责任未知 | 绘制数据流、最小化字段、脱敏、区域与供应商条款审查 |
| 渠道平台审批、模板、限流或主体条件造成延期 | 将每个渠道设为独立里程碑，以单 Inbox 完成 MVP 验收 |
| 直接 AI 回复缺少人工预审 | 仅允许低风险域；高风险使用 Copilot 或直接转人工 |
| 提示注入、工具越权、未验证身份 | 工具白名单、最小权限、响应清洗、身份门、全链路审计 |

## 不在本样例范围内

生产代码修改、生产数据访问、真实客户会话处理、全渠道上线、Chatwoot 私有分支升级或迁移、平台账号/企业主体申请、完整法务合规结论、模型微调、渗透测试、7×24 运维、第三方费用采购、付款或合同承诺。以上事项需要独立授权、证据和报价。
